Chapter 5:Programming Language

In order to facilitate communication between the user and the computer, a programming language is required. This language allows the user to give instructions to the computer to perform specific tasks. Programming languages are composed of a variety of elements, including keywords, variables, operators, loops, and other constructs using various character sets, such as numbers, special symbols, and alphanumeric values. The act of writing a program in a programming language is known as programming, and the person who writes the program is called a programmer.

Programming languages can be categorized into low-level and high-level languages. Low-level languages are machine-dependent, which means that programs written for one system might not work on another. These languages, such as machine-level language (1GL) and assembly language (2GL), require developers to have in-depth knowledge of the system architecture. While these languages offer higher execution speed and direct interaction with hardware, they are complex, time-consuming, and prone to errors. On the other hand, high-level languages are closer to human languages, making them easier to use, write, and debug. Examples of high-level languages include C, C++, Python, and JavaScript, which are machine-independent and allow for faster program development.

There are various types of high-level languages, including procedural-oriented languages (3GL) and object-oriented languages (4GL). Procedural-oriented languages focus on procedures or functions and are used to express the logic of the program. Object-oriented languages, such as Java, C#, and C++, focus on data and allow for greater data security through features like data encapsulation and inheritance. Additionally, natural language processing languages (5GL) are in development, aiming to make programming even more intuitive by allowing users to write statements that resemble everyday language. Programming languages, regardless of their type, require language translators like compilers and interpreters to convert code into machine-readable instructions. These translators facilitate the process of debugging and executing programs efficiently.

Types of Programming Languages

Type of Language Description Examples Advantages Disadvantages
Low-Level Language Machine-dependent languages that interact directly with the hardware. They require detailed knowledge of the system. Machine Level Language (1GL), Assembly Language (2GL) Faster execution, direct hardware control, minimal translation needed. Complex, time-consuming, machine-dependent, difficult to debug.
High-Level Language Closer to human language, machine-independent, easy to write and debug. C, C++, Python, JavaScript Easier to write, machine-independent, less debugging required. Requires a compiler/interpreter, slower execution compared to low-level languages.
Procedural-Oriented Language (3GL) Focuses on the procedure or function of a program. Expresses the logic of the program step by step. C, FORTRAN, QBasic Easier to debug than low-level languages, machine-independent, general-purpose. Less security, slower execution, complex for large programs.
Object-Oriented Language (4GL) Focuses on data and objects, with features like data encapsulation and inheritance for better data security. Java, C#, C++ More secure, allows for better data manipulation, machine-independent. Slower execution, requires more resources, harder to develop hardware-oriented programs.
Natural Language (5GL) Languages resembling everyday language, making programming easier to understand and use. PROLOG Intuitive programming, easier debugging, machine-independent. Still under development, slower execution, not suitable for hardware-oriented tasks.

As programming languages evolve, each type serves different needs, from low-level languages that maximize performance to high-level languages that prioritize ease of use. The progression toward object-oriented and natural languages reflects the ongoing efforts to make programming more accessible and efficient for developers, and to enhance security and performance. The choice of language depends on the complexity of the task, system requirements, and the developer's expertise.
